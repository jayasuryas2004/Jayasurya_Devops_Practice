# AWS EKS IRSA Implementation: Secure DynamoDB Access

## 🚀 Overview
This project demonstrates how to securely connect a Python application running inside an Amazon EKS (Elastic Kubernetes Service) cluster to an AWS DynamoDB table. 

Instead of using hardcoded AWS access keys (which is a major security risk), this architecture utilizes **IAM Roles for Service Accounts (IRSA)**. By establishing an OIDC (OpenID Connect) trust relationship between the EKS cluster and AWS IAM, Kubernetes Pods dynamically receive temporary, least-privilege credentials to access AWS resources.

## 🏗️ Architecture Flow
1. **AWS EKS Cluster:** Hosts the application.
2. **OIDC Provider:** Links the Kubernetes cluster's identity to AWS IAM.
3. **IAM Role & Trust Policy:** An AWS IAM role configured to trust a specific Kubernetes ServiceAccount.
4. **Kubernetes ServiceAccount:** Annotated with the AWS IAM Role ARN.
5. **Python Pod:** Mounts the ServiceAccount. The `boto3` library automatically detects the injected web identity token and assumes the IAM role.

## 📖 Step-by-Step Execution Guide

### Phase 0: Spin up the EKS Cluster
*Note: This project assumes you are using Terragrunt to manage your EKS cluster creation.*
1. Navigate to your EKS Terragrunt directory.
2. Apply the configuration to create the cluster:
```bash
   terragrunt apply -auto-approve

    Update your local kubeconfig to talk to the new cluster:

Bash

   aws eks update-kubeconfig --region us-east-1 --name dev-eks-cluster

Phase 1: AWS Infrastructure (DynamoDB & IAM)

    Create the DynamoDB Table:
    Run this AWS CLI command to create the required table:

Bash

   aws dynamodb create-table \
       --table-name TaskTable \
       --attribute-definitions AttributeName=id,AttributeType=S \
       --key-schema AttributeName=id,KeyType=HASH \
       --billing-mode PAY_PER_REQUEST \
       --region us-east-1

    Provision the IAM Role via Terraform:
    Navigate to the Terraform folder and apply the IAM configuration:

Bash

   cd aws-config_terraform
   terraform init
   terraform apply -auto-approve

    Copy the output irsa_role_arn generated by Terraform.

Phase 2: Kubernetes Configuration (The Bridge)

    Create the Service Account:
    Ensure your service-account.yaml has the correct irsa_role_arn in its annotations. Then apply it:

Bash

   cd ../k8s-manifests
   kubectl apply -f service-account.yaml

    Verify the Annotation:
    Ensure the AWS Role ARN is properly attached:

Bash

   kubectl describe sa dynamodb-sa -n default

    Load the Application Code into a ConfigMap:
    Inject the Python code (main.py) and requirements.txt into the cluster:

Bash

   cd ../app
   kubectl create configmap irsa-app-code --from-file=main.py --from-file=requirements.txt -n default

Phase 3: Deploy and Test

    Deploy the Application:
    Navigate back to the manifests folder and launch the deployment:

Bash

   cd ../k8s-manifests
   kubectl apply -f app-deployment.yaml

    Watch the Magic Happen (Check Logs):
    Stream the logs of your application to verify successful database interaction:

Bash

   kubectl logs -l app=irsa-test -f

Expected Output: You should see successful PutItem, GetItem, and UpdateItem messages.
⚠️ Troubleshooting

    Error: no such host when running kubectl:
    Your kubeconfig is outdated. Re-run the aws eks update-kubeconfig command from Phase 0.

    Logs are blank or stuck:
    Python buffers its output in Kubernetes. Give it 30 seconds. If it's still stuck, ensure your Deployment YAML is using the unbuffered flag: python -u /app/main.py.

🧹 Clean Up (Avoid AWS Charges)

When you are finished practicing, tear down the infrastructure in this exact order to prevent orphan resources:

    Delete Kubernetes Resources:

Bash

   cd k8s-manifests
   kubectl delete -f app-deployment.yaml
   kubectl delete -f service-account.yaml
   kubectl delete configmap irsa-app-code -n default

    Destroy AWS IAM (Terraform):

Bash

   cd ../aws-config_terraform
   terraform destroy -auto-approve

    Delete DynamoDB Table:

Bash

   aws dynamodb delete-table --table-name TaskTable --region us-east-1

    Destroy EKS Cluster:
    Navigate to your Terragrunt directory and run:

Bash

   terragrunt destroy -auto-approve
